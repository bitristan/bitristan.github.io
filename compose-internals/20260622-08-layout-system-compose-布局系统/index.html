三、Constraints：约束系统

Section titled “三、Constraints：约束系统”

3.1 设计理念

Section titled “3.1 设计理念”

Constraints 是父节点与子节点之间的通信协议。父节点通过 Constraints 告诉子节点：“你的宽度/高度必须在这个范围内。“子节点在范围内选择自己的尺寸。

3.2 Long 压缩编码

Section titled “3.2 Long 压缩编码”

Constraints 使用一个 Long（64 位）存储四个整数值，通过变长位分配实现高效存储。编码布局为：

两个宽度值存储在固定位置（低 32 位和高 32 位），两个高度值紧随各自宽度之后。这意味着数据不是按 min/max 维度相邻存放的，而是宽度优先、高度紧随的交错布局。

四种位分配方案（由 FocusIndex 决定）：

FocusIndex名称Width 位数Height 位数适用场景
0MaxFocusHeight1318宽度范围小，高度范围大
1MinFocusHeight1516宽度中等，高度中等
2MinFocusWidth1615宽度中等，高度中等
3MaxFocusWidth1813宽度范围大，高度范围小

FocusIndex 的选择由宽度所需的位数决定：框架先计算 maxWidth（或无界时的 minWidth）需要多少位（13/15/16/18），然后通过 bitOffsetToIndex(widthBits - 13) 映射到 FocusIndex。宽度需要的位数越多，分配给宽度的位越多，分配给高度的位越少。这种设计使得 Constraints 可以表示最多 256K（2^18）的单维度范围，同时避免了对象分配。

3.4 特殊值 Infinity

Section titled “3.4 特殊值 Infinity”

Constraints.Infinity 表示无上限（Int.MAX_VALUE）。当 maxWidth 或 maxHeight 为 Infinity 时：

  • hasBoundedWidth / hasBoundedHeight 返回 false
  • 子节点需要自行决定尺寸（不能依赖”填满父节点”）

这是一个重要的设计模式：用无约束来查询子节点的首选尺寸。例如 wrapContent 的实现就是先用无约束测量子节点，再用子节点的尺寸作为固定约束重新测量。

8.2 单次测量约束

Section titled “8.2 单次测量约束”

Compose 的布局系统有一个关键约束：每个子节点在一次测量 pass 中只能被测量一次。

这个约束在 MeasurePassDelegate 中通过 measuredByParent 标志来强制执行。如果一个子节点被测量两次，框架会抛出异常。

为什么有这个约束？

  1. 性能：避免指数级的重复测量
  2. 一致性：确保父节点和子节点对尺寸的理解一致
  3. 可预测性：简化布局算法的设计

如果需要多次测量（如先查询固有尺寸再正式测量），可以使用 SubcomposeLayout 或固有尺寸 API。

12.1 为什么 Constraints 用 Long 压缩？

Section titled “12.1 为什么 Constraints 用 Long 压缩？”

避免对象分配。布局测量是一个极其频繁的操作——每帧可能涉及数百次测量。使用 value class + Long 编码使得 Constraints 可以内联传递，零 GC 压力。

12.2 为什么单次测量约束？

Section titled “12.2 为什么单次测量约束？”

防止测量的指数爆炸。考虑一个简单的 Column：如果允许重复测量，父节点的每次测量都可能导致子节点被测量两次（一次查询高度，一次正式测量），复杂度变为 O(2^n)。

12.3 为什么分离测量和放置？

Section titled “12.3 为什么分离测量和放置？”
  1. 增量更新：只有位置变化时，可以只重新放置而不重新测量
  2. 动画优化：移动布局时只需要更新位置，不需要重新计算尺寸
  3. Lookahead 支持：可以测量一次（得到尺寸），然后分别放置（实际位置和目标位置）

12.4 为什么 PlacementScope 是 abstract class？

Section titled “12.4 为什么 PlacementScope 是 abstract class？”

PlacementScope 需要持有父节点的 width 和 layoutDirection 来支持 placeRelative 的 RTL 镜像。通过 abstract class，不同的上下文（正常测量、Lookahead、测试）可以提供不同的实现。
