4.2 ContentDrawScope

Section titled “4.2 ContentDrawScope”

ContentDrawScope 扩展了 DrawScope，添加了 drawContent() 方法。关键：如果 drawContent() 不被调用，子内容将不会被绘制。这使得绘制 Modifier 可以控制绘制顺序——在子内容之前、之后、或有条件地绘制。

4.6 drawWithCache：缓存绘制

Section titled “4.6 drawWithCache：缓存绘制”

CacheDrawScope 提供了 onDrawBehind() 和 onDrawWithContent() 来定义缓存的绘制逻辑。缓存只在尺寸变化或显式 invalidateDrawCache() 时重新执行，避免了每帧重复创建 Path、Shader 等昂贵对象。

五、LayoutNodeDrawScope：绘制调度器

Section titled “五、LayoutNodeDrawScope：绘制调度器”

5.1 角色

Section titled “5.1 角色”

LayoutNodeDrawScope 是框架内部的绘制调度器，它：

  1. 实现 ContentDrawScope 接口
  2. 管理当前正在绘制的 DrawModifierNode
  3. 负责 drawContent() 的分发
  4. 持有一个 CanvasDrawScope 实例来实际执行绘制命令

5.2 绘制流程

Section titled “5.2 绘制流程”

draw() 从给定的 drawNode 开始，遍历节点链中所有实现了 DrawModifierNode 的节点，依次调用 drawDirect()。

5.3 drawDirect

Section titled “5.3 drawDirect”

关键步骤：

  1. 保存当前 drawNode，设置新的 drawNode
  2. 在 CanvasDrawScope.draw() 中设置绘制环境（Canvas、Size、Density、LayoutDirection）
  3. 调用 DrawModifierNode.draw()
  4. 恢复之前的 drawNode

6.4 线程安全

Section titled “6.4 线程安全”

LayoutNodeDrawScope 在一个布局树中是单例共享的（单线程）。这意味着：

  • Paint 对象可以在整个树中复用
  • 不同线程的布局树需要独立的 LayoutNodeDrawScope 实例

八、GraphicsLayer：离屏渲染与变换

Section titled “八、GraphicsLayer：离屏渲染与变换”

8.1 概念

Section titled “8.1 概念”

GraphicsLayer 是一个显示列表（Display List），它将绘制命令录制到一个独立的渲染缓冲中。核心优势：

  1. 独立变换：位置、缩放、旋转、透明度等变换可以独立于绘制内容更新
  2. 离屏渲染：复杂绘制可以缓存为纹理，避免每帧重绘
  3. 硬件加速：平台可以将层交给 GPU 处理

8.2 使用方式

Section titled “8.2 使用方式”

8.3 属性

Section titled “8.3 属性”

GraphicsLayer 持有以下变换属性：

  • translationX/Y/Z：平移
  • scaleX/Y：缩放
  • rotationX/Y/Z：旋转
  • alpha：透明度
  • shadowElevation：阴影高度
  • ambientShadowColor / spotShadowColor：阴影颜色
  • blendMode：混合模式
  • renderEffect：渲染效果（模糊、色彩矩阵等）

这些属性可以在不重新录制显示列表的情况下修改——只需要更新变换矩阵。

十一、关键设计决策

Section titled “十一、关键设计决策”

11.1 为什么分离 DrawScope 和 Canvas？

Section titled “11.1 为什么分离 DrawScope 和 Canvas？”

DrawScope 提供了高级、类型安全的绘制 API（接受 Color、Brush、Size 等），而 Canvas 提供了低级、命令式的绘制原语。这种分离使得：

  1. 开发者友好：不需要手动管理 Paint 对象
  2. 平台无关：DrawScope 的 API 不依赖平台特定的 Canvas 实现
  3. 可扩展：可以在 DrawScope 层添加新的绘制能力而不影响 Canvas

11.2 为什么需要 GraphicsLayer？

Section titled “11.2 为什么需要 GraphicsLayer？”
  1. 动画性能：位置/缩放/透明度变化不需要重绘内容
  2. 离屏渲染：复杂绘制可以缓存为纹理
  3. 混合模式：需要 saveLayer 的效果可以独立处理
  4. 阴影：阴影渲染需要独立的渲染通道

11.3 为什么 LayoutNodeDrawScope 是单例共享的？

Section titled “11.3 为什么 LayoutNodeDrawScope 是单例共享的？”

每个 LayoutNode 的绘制需要一个 CanvasDrawScope，但创建它是有成本的（管理 Paint 对象等）。通过在布局树中共享一个实例，避免了大量对象分配。这在单线程环境下是安全的。

11.4 为什么 drawContent() 是显式调用？

Section titled “11.4 为什么 drawContent() 是显式调用？”

显式调用 drawContent() 给了绘制 Modifier 完全的控制权：

  • 可以在子内容之前绘制（背景）
  • 可以在子内容之后绘制（前景/遮罩）
  • 可以有条件地绘制子内容
  • 可以多次绘制子内容（镜像效果）

这比隐式调用更灵活。
