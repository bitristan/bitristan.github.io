4.2 kindSet 与 aggregateChildKindSet

Section titled “4.2 kindSet 与 aggregateChildKindSet”

kindSet：本节点实现的接口类型掩码

aggregateChildKindSet：从本节点到链尾所有节点的类型掩码的 OR 值

  • A.aggregateChildKindSet = 0b110 (Layout | Draw)
  • B.aggregateChildKindSet = 0b110 (Layout | Draw)
  • C.aggregateChildKindSet = 0b010 (Layout)

这个聚合掩码用于快速剪枝：遍历节点链时，如果当前节点的 aggregateChildKindSet 不包含目标类型，可以跳过整个子链。

十、关键设计决策

Section titled “十、关键设计决策”

10.1 为什么 Modifier 是不可变的？

Section titled “10.1 为什么 Modifier 是不可变的？”

不可变性使得：

  1. 重组优化：通过 equals 快速判断是否需要更新
  2. 线程安全：可以在任意线程创建 Modifier
  3. 缓存友好：相同的 Modifier 可以安全复用

10.2 为什么需要两层架构（Element + Node）？

Section titled “10.2 为什么需要两层架构（Element + Node）？”
  • Element 是数据：每次重组可能重新创建，用于描述”应该是什么”
  • Node 是行为：长生命周期，持有状态，用于执行”怎么做”

这种分离使得重组时只需要比较轻量的 Element，而不需要重建整个 Node 链。

10.3 为什么用位掩码标识类型？

Section titled “10.3 为什么用位掩码标识类型？”

位掩码使得类型检查变成简单的位运算：

这比 instanceof 检查快得多，特别是在频繁遍历的场景中。

10.4 为什么需要 DelegatingNode？

Section titled “10.4 为什么需要 DelegatingNode？”

组合优于继承。DelegatingNode 允许将多个独立的功能模块组合成一个复合节点，避免了多重继承的问题，同时保持了每个功能模块的独立性。
