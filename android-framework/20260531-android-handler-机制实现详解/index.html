6.5 实际应用——VSync 信号

Section titled “6.5 实际应用——VSync 信号”

Android 的 Choreographer 使用同步屏障确保 VSync 信号优先处理：

  1. VSync 到来时，投递同步屏障
  2. 投递异步的 VSync 回调消息
  3. VSync 回调执行完后，移除屏障
  4. 被阻塞的同步消息恢复正常处理

这确保了绘制相关操作不被普通消息延迟。

11.1 为什么用 epoll + eventfd 而不是简单的 wait/notify？

Section titled “11.1 为什么用 epoll + eventfd 而不是简单的 wait/notify？”
方案问题
Object.wait/notify无法同时监听 fd 事件（Input、Sensor）
pipe需要两个 fd，有内核缓冲区拷贝开销
eventfd + epoll一个 fd，零拷贝，同时监听消息和 fd 事件

11.2 为什么消息队列用链表而不是优先队列？

Section titled “11.2 为什么消息队列用链表而不是优先队列？”
  • 链表按 when 有序，大部分消息插入尾部，O(1)
  • 只有插入队头时需要唤醒，触发 epoll
  • 代码简单，CPU 缓存友好

11.3 为什么需要同步屏障？

Section titled “11.3 为什么需要同步屏障？”

VSync 是实时信号（每 16.6ms 一次），如果队列中有大量同步消息，VSync 回调可能被延迟，导致掉帧。同步屏障确保 VSync 回调（异步消息）优先执行。

11.4 为什么用对象池而不是创建新 Message？

Section titled “11.4 为什么用对象池而不是创建新 Message？”
  • 主线程每秒可能处理数千条消息
  • 频繁创建/销毁导致 GC 压力
  • 对象池最大 50 个，链表实现，获取/回收 O(1)
