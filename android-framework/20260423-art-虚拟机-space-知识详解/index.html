ART 虚拟机 Space 知识详解

一、引言

Section titled “一、引言”

ART (Android Runtime) 的垃圾回收器采用分代、分区域的内存管理策略。Space 是 ART 堆管理的核心抽象，每个 Space 代表一块具有特定分配策略、回收策略的内存区域。

理解 Space 体系是掌握 ART GC 机制的关键。本文将从类结构、各子类特性、与 GC 协作等维度，系统介绍 ART 的 Space 体系。

4.3.2 DlMallocSpace

Section titled “4.3.2 DlMallocSpace”

基于 Doug Lea malloc 实现的分配空间。

特性：

  • 使用 mspace_ (dlmalloc 句柄) 管理内存
  • 不支持线程本地分配 (TLAB)
  • 适合单线程或低并发场景

4.3.3 RosAllocSpace

Section titled “4.3.3 RosAllocSpace”

基于 rosalloc (Runs-of-Slots Allocator) 实现。

特性：

  • 支持线程本地分配 (TLAB)，多线程性能更好
  • 支持 low_memory_mode_，更积极释放空闲页面
  • 分配时自动清零内存

核心实现：

RosAlloc 原理：

将内存划分为多个 “run”，每个 run 包含固定大小的 slot。不同 run 对应不同大小的对象。线程可以从本地 run 快速分配，避免锁竞争。

4.4 BumpPointerSpace — 指针碰撞空间

Section titled “4.4 BumpPointerSpace — 指针碰撞空间”

分配原理： 通过移动指针实现 O(1) 时间分配，无 free 实现。

核心特性：

  • 无 free 实现，设计为被 evacuate (撤离/复制)
  • 支持 TLAB
  • CanMoveObjects() = true

配合 GC：

用于 Semi-Space GC 和 Mark-Compact GC。GC 时将存活对象复制到另一个空间，然后清空原空间。

十、总结

Section titled “十、总结”

10.1 设计思想

Section titled “10.1 设计思想”

ART Space 体系的设计体现了以下思想：

  1. 职责分离：不同空间承担不同职责，便于针对性优化
  2. 策略可配：通过 GC 保留策略控制回收时机
  3. 并发友好：TLAB、Region 等机制减少锁竞争
  4. 分层抽象：基类定义接口，子类实现具体策略

RosAllocSpace应用运行时分配（默认选择）
