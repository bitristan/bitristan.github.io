ARM64 Linux 启动原理（十一）：链接脚本

基于 Linux v6.10 源码，arch/arm64/kernel/vmlinux.lds.S

1. 概述

Section titled “1. 概述”

链接脚本 vmlinux.lds.S 定义了 ARM64 内核镜像的内存布局。它决定了每个 section 的位置、对齐和大小，直接影响启动过程中页表创建、身份映射和 KPTI 的正确性。

2. 关键宏定义

Section titled “2. 关键宏定义”

2.1 HYPERVISOR_TEXT

Section titled “2.1 HYPERVISOR_TEXT”

KVM hypervisor 代码需要独立的页对齐区域。

2.2 IDMAP_TEXT

Section titled “2.2 IDMAP_TEXT”

身份映射代码必须在一页内（4KB），因为它同时被映射到身份映射和内核虚拟地址空间。

2.3 TRAMP_TEXT（KPTI）

Section titled “2.3 TRAMP_TEXT（KPTI）”

KPTI 跳板代码在 .entry.tramp.text section 中，页对齐。

4. 各段详解

Section titled “4. 各段详解”

4.1 .head.text：镜像头部

Section titled “4.1 .head.text：镜像头部”
  • _text = KIMAGE_VADDR（链接脚本 ASSERT 强制）
  • 包含 PE/COFF 头和 b primary_entry 分支指令
  • HEAD_TEXT 宏展开为 *(.head.text)

4.2 .text：内核代码

Section titled “4.2 .text：内核代码”

每个子段通过 xxx_TEXT 宏控制链接位置，某些段有特殊的缓存对齐要求。

4.3 .rodata.text：特殊代码段

Section titled “4.3 .rodata.text：特殊代码段”

这些代码段”不通过内核映射执行”——它们在特殊地址空间（身份映射、KPTI 跳板）中运行。

4.4 页表目录位置

Section titled “4.4 页表目录位置”

这些页表目录连续排列在 .rodata.text 之后。链接脚本 ASSERT 验证它们的相对偏移：

TRAMP_SWAPPER_OFFSET = 2 * PAGE_SIZE，KPTI 通过在 ttbr1_el1 上加减这个值来切换页表。

4.5 初始化段

Section titled “4.5 初始化段”

__init_begin 到 __init_end 之间的所有内容在内核初始化完成后被释放（free_initmem()），回收内存。

init_idmap_pg_dir 位于 __initdata_begin 之后，因为它只在启动时使用，可以被释放。

4.6 .mmuoff.data：MMU 关闭时访问的数据

Section titled “4.6 .mmuoff.data：MMU 关闭时访问的数据”

这些数据在 MMU 关闭时写入、开启时读取（或反过来）。需要特殊的缓存维护：

  • .mmuoff.data.write：MMU 关闭时写入，需要失效缓存行
  • .mmuoff.data.read：MMU 关闭时读取，需要清理缓存行

2KB 对齐确保缓存维护操作不影响相邻数据。

4.7 init_pg_dir 和 early_init_stack

Section titled “4.7 init_pg_dir 和 early_init_stack”

init_pg_dir 是用于临时映射的页表（__pi_early_map_kernel 中使用）。 early_init_stack 是 4KB 的早期栈，在 primary_entry 中设置为 SP，供 C 函数（如 __pi_create_init_idmap）使用。

Section titled “5.2 为什么这些约束重要”
约束原因
_text == KIMAGE_VADDR虚拟-物理地址转换依赖此对齐
idmap_text <= 4KB身份映射只映射一页，超出会导致翻译失败

页表偏移固定KPTI 通过加减固定偏移切换页表
