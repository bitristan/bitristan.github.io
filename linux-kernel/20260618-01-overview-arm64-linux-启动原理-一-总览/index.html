ARM64 Linux 启动原理（一）：总览

基于 Linux v6.10 源码，arch/arm64/ 架构

1. 概述

Section titled “1. 概述”

ARM64 内核有两种启动方式：

  • Direct Boot：Bootloader（U-Boot/GRUB）直接加载 Image，跳转到 primary_entry（head.S）。此时 MMU 和 D-cache 必须关闭。
  • EFI Boot：UEFI 固件将内核视为 PE/COFF 可执行文件，通过 EFI Stub（efi_pe_entry）中转后，最终也跳转到 primary_entry。此时 MMU 和 D-cache 可能已开启。

两条路径殊途同归，都从 primary_entry 进入内核。EFI Stub 的详细流程见第 2 篇，本文聚焦于汇编入口之后的通用路径。

  • 如果从 EL2 启动（裸机/虚拟化场景），init_kernel_el 配置 EL2 状态后 ERET 到 EL1

2.4 位置无关代码（PIC）

Section titled “2.4 位置无关代码（PIC）”

启动早期 MMU 关闭，内核尚未映射到虚拟地址。此时调用的函数必须是位置无关的（Position Independent），通过 __pi_ 前缀标识：

  • __pi_create_init_idmap：创建初始身份映射
  • __pi_early_map_kernel：映射内核到虚拟地址

这些函数不依赖绝对地址，通过 ADRP（页基址相对）+ ADD 指令组合访问数据，而非绝对地址加载。内核通过 image-vars.h 中的 PROVIDE 宏将 __pi_ 符号链接到实际函数。
