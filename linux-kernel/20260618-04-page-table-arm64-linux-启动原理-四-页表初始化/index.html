1. 概述

Section titled “1. 概述”

ARM64 的页表系统是启动过程中最关键的部分之一。从 __cpu_setup 配置寄存器，到 __enable_mmu 开启 MMU，再到 paging_init 建立最终映射，每一步都涉及页表的创建和切换。

本文覆盖：

  • 页表层级结构和地址翻译原理

  • 三个页表目录的用途和生命周期
  • paging_init() 如何建立最终的内核页表

2. 页表层级结构

Section titled “2. 页表层级结构”

2.1 基本结构

Section titled “2.1 基本结构”

ARM64 使用 4 级页表（默认配置），从高到低：

级别索引位数每级条目数映射大小（4K 页）
PGD9 bits512512 GB
P4D9 bits5121 TB（LPA2）
PUD9 bits5121 GB
PMD9 bits5122 MB
PTE9 bits5124 KB

实际级数由 CONFIG_PGTABLE_LEVELS 决定：

  • 4K 页 + 39-bit VA → 3 级（PGD → PMD → PTE）
  • 4K 页 + 48-bit VA → 4 级（PGD → PUD → PMD → PTE）
  • 16K 页 + 47-bit VA → 3 级
  • 64K 页 + 42-bit VA → 2 级

2.2 块映射（Block Mapping）

Section titled “2.2 块映射（Block Mapping）”

除了逐页映射，ARM64 支持大块映射以减少 TLB 压力：

级别映射大小（4K 页）条件
PUD1 GB地址 1GB 对齐
PMD2 MB地址 2MB 对齐
PTE4 KB（连续映射 64KB）地址 64KB 对齐 + PTE_CONT

3.1 MAIR_EL1：内存属性

Section titled “3.1 MAIR_EL1：内存属性”

MAIR_EL1_SET 定义了 8 种内存类型编码，页表项通过 AttrIndx 字段引用：

每个属性由两个 4-bit 字段组成：

  • 高 4 位：Outer 缓存属性
  • 低 4 位：Inner 缓存属性

例如 0xff 表示 Inner/Outer 都是 Write-Back Read-Allocate Write-Allocate。

3.3 动态特性调整

Section titled “3.3 动态特性调整”

__cpu_setup 不是静态配置，而是根据 CPU 特性动态调整：

  1. 52-bit VA（CONFIG_ARM64_VA_BITS_52）：如果 CPU 支持，调整 T1SZ
  2. LPA2（Large Physical Address Extension 2）：设置 TCR_DS 位
  3. 硬件访问标志（ID_AA64MMFR1_EL1.HAFDBS）：设置 TCR_HA
  4. PIE/PIR（Permission Indirection）：如果支持，配置 PIRE0_EL1、PIR_EL1、TCR2_EL1

4. 三个页表目录

Section titled “4. 三个页表目录”

内核启动过程中使用三个页表目录，各有不同用途：

4.1 init_idmap_pg_dir：初始身份映射

Section titled “4.1 init_idmap_pg_dir：初始身份映射”
  • 创建者：__pi_create_init_idmap（head.S 阶段）
  • 用途：开启 MMU 时使用（TTBR0_EL1）
  • 映射：_stext 到 _end，VA == PA
  • 生命周期：启动后由 cpu_uninstall_idmap 替换为 zero page
  • 位置：__initdata_begin 之后，__init_end 之前（init 段，启动后释放）

4.2 swapper_pg_dir：内核主页表

Section titled “4.2 swapper_pg_dir：内核主页表”
  • 创建者：__pi_early_map_kernel（head.S 阶段）+ paging_init()（C 阶段）
  • 用途：内核运行时的主页表（TTBR1_EL1）
  • 映射：
    • 早期：内核代码/数据的虚拟地址映射
    • 最终：完整的内核地址空间（线性映射、vmalloc、内核代码等）
  • 生命周期：永久使用
  • 位置：.rodata.text 段之后

4.3 idmap_pg_dir：运行时身份映射

Section titled “4.3 idmap_pg_dir：运行时身份映射”
  • 创建者：create_idmap()（paging_init 中调用）
  • 用途：CPU 热插拔、kexec 时使用
  • 映射：__idmap_text_start 到 __idmap_text_end，VA == PA
  • 生命周期：永久使用
  • 位置：.rodata.text 段之后

5.2 map_mem()：线性映射

Section titled “5.2 map_mem()：线性映射”

关键点：

  • 线性映射将所有物理内存映射到 PAGE_OFFSET 开始的虚拟地址
  • 内核代码/数据单独映射，避免创建可写的别名

5.4 create_idmap()：运行时身份映射

Section titled “5.4 create_idmap()：运行时身份映射”

运行时身份映射与初始身份映射的区别：

  • 初始身份映射（init_idmap_pg_dir）：映射整个内核 _stext 到 _end，启动后释放
  • 运行时身份映射（idmap_pg_dir）：仅映射 .idmap.text 段，永久保留，用于 CPU 热插拔

7.2 开启 MMU 瞬间的状态

Section titled “7.2 开启 MMU 瞬间的状态”

为什么需要身份映射：MMU 开启的瞬间，PC 中仍然是物理地址。如果没有身份映射，这个地址无法被翻译，CPU 会立即 fault。身份映射确保物理地址被翻译到相同的虚拟地址，CPU 继续执行同一条指令流。
