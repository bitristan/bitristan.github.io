ARM64 Linux 启动原理（七）：start_kernel

基于 Linux v6.10 源码，init/main.c

1. 概述

Section titled “1. 概述”

start_kernel() 是所有架构共享的内核初始化入口，由 __primary_switched（head.S）调用。它按顺序初始化内核的各个子系统，最终通过 rest_init() 创建 PID 1（init 进程）和 PID 2（kthreadd），然后 boot CPU 进入 idle 循环。

2. 函数签名与属性

Section titled “2. 函数签名与属性”
属性含义
asmlinkage从汇编代码调用，使用栈传递参数
__visible防止编译器优化掉符号（LTO 下需要）
__init函数代码在 __init_begin 到 __init_end 之间，启动后释放
__no_sanitize_address禁用 AddressSanitizer（KASAN 此时尚未完全初始化）
__noreturn函数不会返回
__no_stack_protector禁用栈保护（stack canary 此时尚未初始化）

Section titled “5. initcall 机制”

5.1 initcall 级别

Section titled “5.1 initcall 级别”

内核通过链接脚本将 initcall 函数指针分组到不同的 section，按级别顺序执行：

级别宏Section含义
earlyearly_initcall().initcallearly.initSMP 之前执行
0pure_initcall().initcall0.init纯初始化（无依赖）
1core_initcall().initcall1.init核心子系统
1score_initcall_sync().initcall1s.init核心同步
2postcore_initcall().initcall2.init核心之后
3arch_initcall().initcall3.init架构初始化
4subsys_initcall().initcall4.init子系统
5fs_initcall().initcall5.init文件系统
rootfsrootfs_initcall().initcallrootfs.initrootfs（5 和 6 之间）
6device_initcall().initcall6.init设备驱动（默认）
7late_initcall().initcall7.init后期初始化

__initcall(fn) 是 device_initcall(fn) 的别名，因此未指定级别的 initcall 默认为级别 6。
