3. EL2/EL1 切换与 VHE

Section titled “3. EL2/EL1 切换与 VHE”

3.1 VHE（Virtualization Host Extensions）

Section titled “3.1 VHE（Virtualization Host Extensions）”

VHE 允许 Linux 内核直接运行在 EL2，无需额外的 hypervisor 层。这简化了 KVM 的实现，因为内核可以直接管理 Stage-2 页表。

模式内核运行级别用途
nVHEEL1传统模式，EL2 运行 hyp stub
VHEEL2内核直接在 EL2 运行

3.2 init_kernel_el()：启动时的 EL 切换

Section titled “3.2 init_kernel_el()：启动时的 EL 切换”

返回值：

  • BOOT_CPU_MODE_EL1（0xe22）：从 EL1 启动
  • BOOT_CPU_MODE_EL2（0xe12）：从 EL2 启动（已降到 EL1）

3.4 __finalise_el2：EL2 侧的处理

Section titled “3.4 __finalise_el2：EL2 侧的处理”

VHE 升级的关键步骤：

  1. 验证 CPU 支持 VHE（ID_AA64MMFR1_EL1）
  2. 设置 HCR_EL2.E2H 和 HCR_EL2.TGE
  3. 将 EL1 的状态（SP、页表、中断向量）迁移到 EL2
  4. 修改返回地址的 SPSR，使 eret 返回到 EL2 而非 EL1

5. KPTI（内核页表隔离）

Section titled “5. KPTI（内核页表隔离）”

5.1 原理

Section titled “5.1 原理”

KPTI（Kernel Page Table Isolation）是针对 Meltdown 漏洞的缓解措施。核心思想：当 CPU 运行用户态代码时，从地址空间中移除内核映射。

5.4 tramp_map_kernel / tramp_unmap_kernel

Section titled “5.4 tramp_map_kernel / tramp_unmap_kernel”

TRAMP_SWAPPER_OFFSET = 2 * PAGE_SIZE。由于 tramp_pg_dir 和 swapper_pg_dir 在内存中相邻且偏移固定，页表切换只需一条 ADD/SUB 指令。

5.5 退出路径跳板

Section titled “5.5 退出路径跳板”

tramp_exit 执行 tramp_unmap_kernel（移除内核映射），然后 eret 返回用户态。

5.6 KPTI 性能影响

Section titled “5.6 KPTI 性能影响”
  • TLB 刷新：每次进出内核都需要切换页表，导致 TLB flush
  • PCID 缓解：如果 CPU 支持 ASID/PCID，可以减少 TLB flush 的开销
  • KPTI 默认开启：CONFIG_UNMAP_KERNEL_AT_EL0 默认 y

8. MTE（Memory Tagging Extension）

Section titled “8. MTE（Memory Tagging Extension）”

8.1 原理

Section titled “8.1 原理”

MTE 是 ARMv8.5 引入的内存安全特性，为每个 16 字节粒度的内存分配一个 4-bit 标签。指针的高位也包含标签。访问时比较指针标签和内存标签，不匹配则触发异常。

9. UEFI Runtime Services

Section titled “9. UEFI Runtime Services”

9.1 初始化

Section titled “9.1 初始化”

efi_init() 在 setup_arch() 中调用，完成：

  1. 从 FDT 中的 /chosen/linux,uefi-system-table 获取 UEFI 系统表地址
  2. 获取 UEFI 内存映射
  3. 根据 efi_novamap 决定是否调用 SetVirtualAddressMap()

9.2 efi_novamap 逻辑

Section titled “9.2 efi_novamap 逻辑”
  • VA_BITS_MIN >= 48：内核可以直接 1:1 映射 UEFI 运行时区域
  • system_needs_vamap()：某些处理器（如 Ampere）需要 SetVirtualAddressMap()
