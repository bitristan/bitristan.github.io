ARM64 Linux 启动原理（五）：设备树解析

基于 Linux v6.10 源码，arch/arm64/ 架构

1. 概述

Section titled “1. 概述”

Device Tree（设备树）是 ARM Linux 描述硬件拓扑的核心机制。在启动过程中，FDT（Flattened Device Tree）经历了三个阶段：

  1. 传递：Bootloader 将 FDT 物理地址放入 x0，内核在 head.S 中保存到 x21 和 boot_args[0]
  2. 解析：setup_arch() 通过 fixmap 将 FDT 映射到虚拟地址，提取启动参数和内存布局
  3. 转化：unflatten_device_tree() 将扁平的 FDT 二进制数据转化为内核运行时使用的 device_node 树

4.1 为什么需要 fixmap

Section titled “4.1 为什么需要 fixmap”

FDT 位于物理内存中，setup_arch() 执行时内核的线性映射尚未建立（paging_init() 还未调用）。fixmap 提供了一种在页表完全初始化之前映射物理地址的机制。

8. FDT 的内存保护

Section titled “8. FDT 的内存保护”

FDT 在启动过程中经历了权限变化：

阶段映射属性原因
setup_machine_fdt 首次映射PAGE_KERNEL（读写）早期扫描可能修改 FDT
early_init_dt_scan 完成后PAGE_KERNEL_RO（只读）防止意外修改
memblock_reserve物理内存保留防止被分配给其他用途

9. FDT 在内核镜像中的位置

Section titled “9. FDT 在内核镜像中的位置”

FDT 可以通过两种方式提供：

  1. 外部 FDT：Bootloader 加载独立的 .dtb 文件，将地址传入 x0
  2. 内嵌 FDT：通过 CONFIG_ARM64_APPENDED_DTB 或 DTB 内嵌到内核镜像

外部 FDT 是最常见的场景，也是 ARM64 标准 boot protocol 的要求。
