ARM64 Linux 启动原理（六）：setup_arch

基于 Linux v6.10 源码，arch/arm64/kernel/setup.c

1. 概述

Section titled “1. 概述”

setup_arch() 是 ARM64 架构初始化的核心函数，在 start_kernel() 中调用。它完成了从硬件探测到内存管理子系统建立的全部工作，是连接汇编启动阶段和通用 C 内核的桥梁。

4. kaslr_init()：内核地址随机化

Section titled “4. kaslr_init()：内核地址随机化”

KASLR 在两个阶段工作：

阶段执行时机功能
早期__pi_early_map_kernel（head.S）通过 FDT kaslr-seed 计算虚拟地址偏移，映射内核
初始化kaslr_init()（此处）记录 kaslr_offset 到全局变量

KASLR 偏移的影响：

  • 内核实际虚拟基址 = KIMAGE_VADDR + kaslr_offset
  • kimage_voffset = 虚拟地址 - 物理地址（在 __primary_switched 中计算）

Section titled “5. early_fixmap_init()：fixmap 早期页表”

fixmap 使用静态分配的页表项（bm_pte、bm_pmd、bm_pud），不依赖动态内存分配：

关键约束（fixmap.c:27）：static_assert(NR_BM_PMD_TABLES == 1) — fixmap 区域不超过一个 PUD 条目。

6. setup_machine_fdt()：FDT 映射与扫描

Section titled “6. setup_machine_fdt()：FDT 映射与扫描”

详见第 5 篇 — Device Tree。核心流程：

  1. fixmap_remap_fdt() 通过 fixmap 映射 FDT
  2. early_init_dt_scan() 扫描 /chosen、/memory 等节点
  3. 提取命令行、内存布局、initrd 位置等信息
  4. 重新映射 FDT 为只读

7. 异常状态配置与身份映射移除

Section titled “7. 异常状态配置与身份映射移除”

DAIF 标志位控制四种异常：

标志含义此时状态原因
DDebug取消屏蔽允许调试
ASError取消屏蔽允许系统错误报告
IIRQ保持屏蔽GIC 尚未初始化
FFIQ保持屏蔽GIC 尚未初始化

将 TTBR0_EL1 指向 zero page（reserved_pg_dir），移除启动时的身份映射。运行时身份映射（idmap_pg_dir）已由 paging_init() 创建。
