ARM64 Linux 启动原理（九）：SMP 多核启动

基于 Linux v6.10 源码，arch/arm64/kernel/smp.c、smp_spin_table.c、psci.c

1. 概述

Section titled “1. 概述”

ARM64 SMP（Symmetric Multi-Processing）启动将 CPU 从初始的单核状态扩展到多核并行运行。启动过程涉及：

  1. CPU 枚举：从 DT 或 ACPI 获取所有 CPU 的 MPIDR
  2. cpu_ops 选择：根据 DT enable-method 选择启动方法
  3. 从核唤醒：通过 spin-table 或 PSCI 唤醒从核
  4. 从核初始化：从核建立页表、进入内核、报告就绪
  5. IPI 配置：核间中断机制建立

6. PSCI 方法

Section titled “6. PSCI 方法”

6.1 原理

Section titled “6.1 原理”

PSCI（Power State Coordination Interface）是 ARM 标准的固件接口，通过 SMC 或 HVC 调用固件管理 CPU 电源状态。

PSCI 函数编号功能
CPU_ON0x84000003启动指定 CPU
CPU_OFF0x84000002关闭当前 CPU
CPU_SUSPEND0x84000001挂起当前 CPU
SYSTEM_RESET0x84000009系统重置

6.2 psci_cpu_boot() 实现

Section titled “6.2 psci_cpu_boot() 实现”

CPU_ON 的参数：

  • target_cpu：目标 CPU 的 MPIDR
  • entry_point：目标 CPU 的入口物理地址（secondary_entry）

固件将目标 CPU 从 power-off 状态唤醒，使其从 secondary_entry 开始执行。
