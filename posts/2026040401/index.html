    Android VSYNC 机制深度解析

    概述

    VSYNC（垂直同步信号）是 Android 图形系统的核心机制，用于协调显示刷新与渲染时机。本文从源码层面深入分析 VSYNC 的产生、传递和处理全过程。

    // HWComposer.cpp:156-185
std::optional<PhysicalDisplayId> HWComposer::onVsync(hal::HWDisplayId hwcDisplayId,
                                                      nsecs_t timestamp) {
    const auto displayIdOpt = toPhysicalDisplayId(hwcDisplayId);
    if (!displayIdOpt) {
        LOG_HWC_DISPLAY_ERROR(hwcDisplayId, "Invalid HWC display");
        return {};
    }

    auto& displayData = mDisplayData[*displayIdOpt];

    // 过滤重复的 VSYNC 事件
    if (timestamp == displayData.lastPresentTimestamp) {
        ALOGW("Ignoring duplicate VSYNC event from HWC");
        return {};
    }

    displayData.lastPresentTimestamp = timestamp;
    
    // Systrace 标记
    SFTRACE_INT(ftl::Concat("HW_VSYNC_", displayIdOpt->value).c_str(),
                displayData.vsyncTraceToggle);
    
    return displayIdOpt;
}

    三、软件 VSYNC 生成机制

    3.1 为什么需要软件 VSYNC？

    硬件 VSYNC 有以下局限：

    1. 功耗问题：持续开启硬件 VSYNC 会增加功耗
    2. 灵活性不足：无法灵活调整唤醒时机
    3. 精度问题：需要预测下一帧时间

    软件 VSYNC 通过预测模型生成，减少对硬件 VSYNC 的依赖。

    // VSyncReactor.cpp:189-210
bool VSyncReactor::addHwVsyncTimestamp(nsecs_t timestamp, 
                                        std::optional<nsecs_t> hwcVsyncPeriod,
                                        bool* periodFlushed) {
    if (periodConfirmed(timestamp, hwcVsyncPeriod)) {
        // 周期确认，关闭硬件 VSYNC
        mLastHwVsync.reset();
        mTracker.addVsyncTimestamp(timestamp);
        *periodFlushed = true;
    } else {
        // 需要继续采集硬件 VSYNC 样本
        mLastHwVsync = timestamp;
    }
    
    // 返回是否需要继续硬件 VSYNC
    return mTracker.needsMoreSamples();
}

bool VsyncSchedule::addResyncSample(TimePoint timestamp, 
                                     ftl::Optional<Period> hwcVsyncPeriod) {
    std::lock_guard<std::mutex> lock(mHwVsyncLock);
    
    if (mHwVsyncState == HwVsyncState::Enabled) {
        needsHwVsync = mController->addHwVsyncTimestamp(
            timestamp.ns(), hwcVsyncPeriod.transform(&Period::ns), &periodFlushed);
    }
    
    if (!needsHwVsync) {
        disableHardwareVsync(false);  // 关闭硬件 VSYNC
    }
    
    return needsHwVsync;
}

    // VSyncDispatchTimerQueue.cpp:310-360
void VSyncDispatchTimerQueue::setTimer(nsecs_t targetTime, nsecs_t now) {
    mIntendedWakeupTime = targetTime;
    // 设置 timerfd 定时器
    mTimeKeeper->alarmAt(
        std::bind(&VSyncDispatchTimerQueue::timerCallback, this),
        mIntendedWakeupTime
    );
}

void VSyncDispatchTimerQueue::timerCallback() {
    // 定时器触发时，执行所有到期的回调
    for (auto& callback : mCallbacks) {
        if (callback->wakeupTime() <= now) {
            callback->callback(vsyncTimestamp, wakeupTimestamp, deadlineTimestamp);
        }
    }
    
    // 重新设置下一个定时器
    rearmTimer(mTimeKeeper->now());
}

    // EventThread.cpp:381-440
void EventThread::threadMain(std::unique_lock<std::mutex>& lock) {
    while (mState != State::Quit) {
        std::vector<sp<EventThreadConnection>> connections;
        
        // 收集需要 VSYNC 的连接
        for (const auto& connection : mDisplayEventConnections) {
            if (connection->vsyncRequest != VSyncRequest::None) {
                connections.push_back(connection);
            }
        }
        
        // 等待 VSYNC 信号
        mCondition.wait(lock);
        
        // 分发 VSYNC 事件
        for (const auto& connection : connections) {
            connection->postEvent(vsyncEvent);
        }
    }
}

    十一、总结

    Android VSYNC 机制是一个多层次、高效率的同步系统：

    1. 硬件层：Display 通过 HWC 产生 VSYNC 信号
    2. 预测层：VSyncPredictor 基于历史数据预测 VSYNC 时间，减少硬件依赖
    3. 分发层：VSyncDispatch 通过定时器精确分发回调
    4. 应用层：Choreographer 协调 App 的绘制时机

    这种设计实现了：

    • 低功耗：仅在必要时开启硬件 VSYNC
    • 高精度：通过预测模型实现精确的帧同步
    • 灵活性：相位偏移允许 App 和 SF 错峰执行
    • 可扩展：支持多种刷新率动态切换
