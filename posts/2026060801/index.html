    Compose 核心概念：Group 与 Scope 详解

    概述

    在 Jetpack Compose 的运行时实现中，Group 和 Scope 是两个核心概念。它们是 Compose 实现增量更新和智能重组的基础。本文将深入讲解这两个概念的工作原理。

    一、Group：SlotTable 的基本单元

    1.1 什么是 Group？

    Group 是 SlotTable 中存储 Composable 执行位置信息的基本单元。每个 Group 在 SlotTable 的 groups 数组中占用 5 个 Int：

    1
2
3

    // SlotTable.kt:3892-3896
//  0             | 1             | 2             | 3             | 4             |
//  Key           | Group info    | Parent anchor | Size          | Data anchor   |
    字段说明
    Key编译器生成的位置标识（基于源码位置）
    Group info位域：存储 isNode/hasObjectKey/hasAux 等标志 + nodeCount
    Parent anchor父 Group 的索引
    Size该 Group 占用的子 Group 数量
    Data anchor指向 slots 数组中的数据位置

    @Composable
fun Counter(count: Int, $composer: Composer, $changed: Int) {
    // 1. 开始 RestartGroup
    $composer.startRestartGroup(key)
    
    // 2. 检查是否可以跳过
    var dirty = $changed
    if ($composer.changed(count)) dirty = dirty or 0b1
    
    if (dirty == 0 && !$composer.skipping) {
        $composer.skipToGroupEnd()  // 跳过执行
    } else {
        Text("Count: $count", $composer, 0)  // 执行函数体
    }
    
    // 3. 结束 RestartGroup，注册重组 lambda
    $composer.endRestartGroup()?.updateScope { composer, _ ->
        Counter(count, composer, 0b1)
    }
}

    $composer.startRestartGroup(key)

    if (showText) {

}

    为什么条件分支需要 Group？

    当 showText 从 true 变为 false 时，Composer 需要知道：

    • 之前有一个 Group（Text 节点）
    • 现在没有了
    • 需要删除这个 Group 及其关联的节点

    ReplaceGroup 标记了这段代码的边界，使得 Composer 可以正确处理内容的添加和删除。

inline fun <T : Any, reified E : Applier<T>> ComposeNode(
    noinline factory: () -> T,

    二、Scope：独立重组的最小单元

    2.1 什么是 Scope？

    Scope（RecomposeScope）是 Compose 中可以独立重组的最小单元。每个可重组的 Composable 函数都会创建一个 Scope，用于：

    1. 记录该函数内读取了哪些状态
    2. 存储如何重新执行该函数的 lambda
    3. 管理该函数的失效/重组状态

    // RecomposeScopeImpl.kt
internal class RecomposeScopeImpl(internal var owner: RecomposeScopeOwner?) :
    ScopeUpdateScope, RecomposeScope {
    
    /** 指向 SlotTable 中的位置 */
    var anchor: Anchor? = null
    
    /** 重组时要执行的 lambda */
    private var block: ((Composer, Int) -> Unit)? = null
    
    /** 是否使用了状态（读取了 mutableState） */
    var used: Boolean = false
    
    /** 是否需要重组 */
    var requiresRecompose: Boolean = false
    
    /** 记录读取过的状态实例 */
    private var trackedInstances: MutableObjectIntMap<Any>? = null
    
    /** 执行重组 */
    fun compose(composer: Composer) {
        block?.invoke(composer, 1) ?: error("Invalid restart scope")
    }
    
    /** 使 scope 失效，触发重组 */
    override fun invalidate() {
        owner?.invalidate(this, null)
    }
    
    /** 更新重组 lambda */
    override fun updateScope(block: (Composer, Int) -> Unit) {
        this.block = block
    }
}

    // GapComposer.kt:2102-2144
override fun startRestartGroup(key: Int): Composer {
    startReplaceGroup(key)
    addRecomposeScope()  // 创建 Scope
    return this
}

private fun addRecomposeScope() {
    if (inserting) {
        // 首次 composition：创建新的 Scope
        val scope = RecomposeScopeImpl(composition as CompositionImpl)
        invalidateStack.push(scope)
        updateValue(scope)  // 存入 SlotTable 的 slots 数组
        enterRecomposeScope(scope)
    } else {
        // 重组：从 SlotTable 读取已有的 Scope
        val slot = reader.next()
        val scope = slot as RecomposeScopeImpl
        invalidateStack.push(scope)
        enterRecomposeScope(scope)
    }
}

    四、总结

    Group 与 Scope 的对比

    概念GroupScope
    定义SlotTable 的基本存储单元可独立重组的最小单元
    创建时机每个 startXxxGroup()仅 startRestartGroup()
    数量关系多个 Group 可能对应一个 Scope每个 Scope 对应一个 RestartGroup
    主要作用标记代码执行位置，组织数据管理重组范围，记录状态订阅
    存储位置groups 数组（5 个 Int）slots 数组（第一个 slot）

    Group 的本质

    1. 位置标识：在 SlotTable 中标记代码执行位置
    2. 状态分组：将相关的 remember、状态等数据组织在一起
    3. 增量更新：通过对比 Group key 决定复用/删除/插入
    4. 生命周期管理：Group 创建时初始化，删除时清理

    Scope 的本质

    1. 重组边界：定义可以独立重组的代码范围
    2. 状态追踪：记录该范围内读取的所有状态
    3. 重组触发：当订阅的状态变化时，精确触发对应的 Scope 重组
    4. 性能优化：避免全局重组，只更新受影响的部分

    不是每个 @Composable 都生成 Group

    情况是否生成 Group生成什么 Group
    可重组函数✅RestartGroup
    inline 函数❌无（被内联展开）
    条件表达式✅ReplaceGroup
    key() 循环✅MovableGroup
    ComposeNode✅Node Group
