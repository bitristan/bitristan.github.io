    ART 虚拟机 Space 知识详解

    一、引言

    ART (Android Runtime) 的垃圾回收器采用分代、分区域的内存管理策略。Space 是 ART 堆管理的核心抽象，每个 Space 代表一块具有特定分配策略、回收策略的内存区域。

    理解 Space 体系是掌握 ART GC 机制的关键。本文将从类结构、各子类特性、与 GC 协作等维度，系统介绍 ART 的 Space 体系。

    4.3.3 RosAllocSpace

    基于 rosalloc (Runs-of-Slots Allocator) 实现。

    特性：

    • 支持线程本地分配 (TLAB)，多线程性能更好
    • 支持 low_memory_mode_，更积极释放空闲页面
    • 分配时自动清零内存

    核心实现：

     1

// 线程本地分配

    RosAlloc 原理：

    将内存划分为多个 “run”，每个 run 包含固定大小的 slot。不同 run 对应不同大小的对象。线程可以从本地 run 快速分配，避免锁竞争。

    4.4 BumpPointerSpace — 指针碰撞空间

    分配原理： 通过移动指针实现 O(1) 时间分配，无 free 实现。

    1
2
3
4
5
6
7

    mirror::Object* AllocNonvirtual(size_t num_bytes) {
  uint8_t* old_end = end_.load(std::memory_order_relaxed);
  uint8_t* new_end = old_end + num_bytes;
  if (new_end > growth_end_) return nullptr;
  end_.store(new_end, std::memory_order_relaxed);
  return reinterpret_cast<mirror::Object*>(old_end);
}

    核心特性：

    • 无 free 实现，设计为被 evacuate (撤离/复制)
    • 支持 TLAB
    • CanMoveObjects() = true

    数据结构：

    1
2
3
4
5

    uint8_t* growth_end_;                    // 增长边界
AtomicInteger objects_allocated_;        // 已分配对象数
AtomicInteger bytes_allocated_;          // 已分配字节数
size_t main_block_size_ GUARDED_BY(lock_);  // 主块大小
std::deque<size_t> block_sizes_ GUARDED_BY(lock_);  // TLAB 块大小队列

    配合 GC：

    用于 Semi-Space GC 和 Mark-Compact GC。GC 时将存活对象复制到另一个空间，然后清空原空间。

    9.3 常见问题与调优

    碎片问题：

    • 使用 Trim() 归还空闲页面
    • 选择支持整理的 GC 算法
    • 调整大对象阈值

    TLAB 调优：

    • 增大 TLAB 大小减少全局分配
    • GC 时及时撤销 TLAB

    十、总结

    10.1 设计思想

    ART Space 体系的设计体现了以下思想：

    1. 职责分离：不同空间承担不同职责，便于针对性优化
    2. 策略可配：通过 GC 保留策略控制回收时机
    3. 并发友好：TLAB、Region 等机制减少锁竞争
    4. 分层抽象：基类定义接口，子类实现具体策略

    10.2 各 Space 适用场景总结

    Space最佳场景
    ImageSpace预编译类、启动优化
    ZygoteSpace进程共享、减少内存占用
    RosAllocSpace应用运行时分配（默认选择）
    BumpPointerSpace复制 GC 的 from/to 空间
    RegionSpaceConcurrent Copying GC
    LargeObjectSpace大数组、大字符串
