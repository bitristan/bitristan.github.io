    Android ART 虚拟机启动流程深度解析

    Android ART 虚拟机启动流程深度解析

    ART（Android Runtime）是 Android 系统从 Dalvik 虚拟机迭代而来的核心运行时组件，负责 Java/Kotlin 代码的编译、解释与执行，是连接 Native 层与 Java 层的关键枢纽。

    其启动流程贯穿 Native 层初始化、内核创建、JNI 通信绑定到 Java 世界启动的全链路，涉及多个核心模块与关键函数的协同工作。本文将从技术视角出发，分阶段拆解 ART 虚拟机的启动流程，剖析各环节的核心作用与底层逻辑。

    一、ART 虚拟机的核心定位与启动前提

    ART 虚拟机并非独立启动，而是依托 Zygote 进程（Android 所有应用进程与系统服务进程的"母体"）完成启动。

    Zygote 进程由系统 init 进程孵化，其核心使命是：

    • 初始化 ART 虚拟机
    • 预加载系统资源
    • 通过 fork 机制高效孵化子进程，实现资源共享与启动加速

    ART 启动的核心目标是：

    创建全局唯一的虚拟机实例，初始化内存管理、类加载、线程调度等核心模块，打通 Native 层与 Java 层的通信通道，最终为 SystemServer 启动、应用进程执行提供稳定的运行环境。

    二、ART 虚拟机启动全流程拆解

    ART 启动流程整体分为 4 个核心阶段，各阶段层层递进、环环相扣。

    阶段一：Native 层基础环境初始化（启动入口）

    此阶段为 ART 虚拟机启动的前置准备，主要完成进程环境初始化、ART 核心库加载，为后续虚拟机内核创建铺路。

    1. main()：Zygote 进程 Native 入口

    负责初始化进程基础环境，包括：

    • 设置进程权限
    • 清理文件描述符
    • 初始化日志系统

    随后触发 AndroidRuntime 实例的初始化，正式开启 ART 虚拟机的启动流程。

    2. JniInvocation::Init(library)

    核心作用是动态加载 ART 虚拟机核心库 libart.so，通过 dlsym 系统调用查找并绑定 JNI_CreateJavaVM、JNI_GetDefaultJavaVMInitArgs 等核心函数指针。

    3. AndroidRuntime::startVm()

    组装 ART 虚拟机的启动参数，包括：

    • JNI 版本
    • 堆内存大小（-Xmx）
    • GC 策略
    • 编译模式（JIT/AOT）

    封装为 JavaVMInitArgs 结构体，最终调用 JNI_CreateJavaVM 函数，触发 ART 虚拟机内核的创建。

    阶段二：ART 虚拟机内核创建与启动（核心阶段）

    此阶段是 ART 虚拟机"诞生"的核心，完成全局唯一的 Runtime 单例创建、核心模块初始化、主线程绑定。

    1. JNI_CreateJavaVM()

    作为 libart.so 导出的 JNI 标准入口函数，是 ART 启动的核心枢纽。主要工作包括：

    • 校验 JNI 版本合法性
    • 加全局锁保证虚拟机单例
    • 解析启动参数并创建 Runtime 实例
    • 初始化主线程并绑定 JNIEnv
    • 启动虚拟机内核

    最终返回 JavaVM（虚拟机全局句柄）和 JNIEnv（主线程 JNI 执行环境）。

    2. Runtime::Create()

    创建 ART 全局单例 Runtime（虚拟机内核），初始化内核所有核心模块：

    模块职责
    Heap内存堆管理，负责内存分配、回收与碎片整理
    ClassLinker类加载与解析，负责加载系统类、应用类，生成 ArtMethod
    ThreadList线程管理，负责线程创建、调度与生命周期管理
    JITCompiler即时编译，将字节码动态编译为机器码
    InternTable字符串常量池，管理全局字符串
    SignalCatcher信号处理，处理系统信号（如崩溃、GC 触发）

    3. Thread::Init()

    初始化 Zygote 主线程：

    • 将主线程与 Runtime 实例关联
    • 绑定 JNIEnvExt（ART 对 JNIEnv 的扩展实现）
    • 设置线程本地存储（TLS）
    • 将主线程加入 ThreadList 进行统一管理

    4. Runtime::Start()

    启动 ART 虚拟机内核：

    • 开启各类后台线程（GC 守护线程、JIT 编译线程、信号处理线程）
    • 预加载系统核心类（如 java.lang.Object、java.lang.String）
    • 完成 JIT/AOT 编译器初始化

    使虚拟机正式进入可运行状态。

    阶段三：系统 JNI 方法注册（打通 Native 与 Java 通信）

    ART 虚拟机启动后，需完成 Native 层与 Java 层的通信绑定——将系统底层 C/C++ 函数与 Java 层 native 方法一一关联。

    1. AndroidRuntime::StartReg()

    JNI 方法注册的统一入口，调用两个核心注册函数。

    2. register_android_methods()

    注册 Android Framework 层专属 JNI 方法，包括：

    • Binder 跨进程通信
    • View 渲染
    • Activity 生命周期管理

    其本质是一个分发函数，汇总调用所有 Framework 子模块的注册函数，最终通过 RegisterNatives 完成绑定。

    3. register_jni_procs()

    注册 Java 核心库（libcore）的 JNI 方法，包括 java.lang.Object、java.lang.Thread、java.io 等基础类的 native 方法。

    通过遍历静态映射表 gRegJNIRecord，批量查找 Java 类并调用 RegisterNatives 完成绑定。

    4. env->RegisterNatives()

    JNI 注册的底层实现：

    1. 根据 Java 类名、方法名与方法签名
    2. 找到已加载类的 ArtMethod 结构体
    3. 将 C/C++ 函数指针写入 ArtMethod 的入口指针

    完成 native 方法与底层函数的硬绑定，后续 Java 调用 native 方法时可直接跳转到对应 C/C++ 函数执行。

    阶段四：进入 Java 世界，完成 Zygote 初始化（启动闭环）

    JNI 方法注册完成后，ART 虚拟机已具备执行 Java 代码的能力，通过 JNI 调用进入 Java 世界。

    1. AndroidRuntime::start()

    通过 JNI 调用 Java 层入口函数 ZygoteInit.main(String[])，将启动流程从 Native 层切换到 Java 层。

    2. ZygoteInit.main(String[])

    Zygote 进程 Java 层的核心入口，完成 Zygote 的最终初始化：

    • 创建 Zygote 服务端 Socket（名称为 zygote），作为 ActivityManagerService（AMS）请求孵化应用进程的通信通道
    • 执行预加载操作，批量加载系统公共资源
    • 孵化 SystemServer 进程，启动系统核心服务
    • 进入无限死循环，监听 AMS 发来的应用进程孵化请求

    3. Zygote 预加载相关函数

    预加载是 Zygote 进程的核心优化手段，通过一次性加载系统公共资源，让所有子进程通过 fork 的写时复制（COW）机制共享内存。

    函数作用
    ZygoteInit.preloadClasses()读取 frameworks/base/preloaded-classes 配置文件，预加载系统核心 Java 类
    ZygoteInit.preloadResources()读取 frameworks/base/core/res/res/values/arrays.xml 配置，预加载系统 Drawable、颜色等资源
    ZygoteInit.preloadSharedLibraries()预加载系统核心共享库（如 libandroid_runtime.so、libgui.so）
    ZygoteInit.preloadGraphics()预加载系统字体（/system/fonts/）、OpenGL 渲染引擎等图形资源

    4. ZygoteInit.forkSystemServer()

    通过 fork 机制孵化 SystemServer 进程（Android 系统的"大脑"）：

    1. 为其配置专属的 UID、GID 与进程权限
    2. 关闭 Zygote 服务端 Socket
    3. 启动 SystemServer.main()

    后续 SystemServer 会启动 AMS、WMS、PMS 等所有核心系统服务，完成 Android 系统的启动。

    5. ZygoteServer.runSelectLoop()

    Zygote 进程进入无限死循环，监听 AMS 发来的应用进程孵化请求。

    收到请求后：

    1. 通过 fork 孵化新的应用进程
    2. 为应用进程配置专属环境（如进程名、应用沙盒）
    3. 启动应用的 ActivityThread.main()
    4. 应用进程正式启动

    Zygote 则继续循环等待下一个请求，完成 ART 启动流程的闭环。

    三、核心总结与关键特性

    ART 虚拟机的启动是"从 Native 到 Java、从初始化到可用“的分层递进过程：

    加载核心库 → 创建虚拟机内核 → 注册 JNI 通信 → 初始化 Zygote → 孵化系统与应用进程

    关键特性

    1. 单例设计

    Runtime 全局唯一，确保进程内只有一个虚拟机实例，避免资源冲突。

    2. 预加载机制

    Zygote 一次性预加载系统资源，子进程共享内存，提升启动效率与内存利用率。

    3. JNI 绑定

    通过 RegisterNatives 硬绑定 Java native 方法与 C/C++ 函数，实现高效通信。

    4. fork 优化

    基于写时复制机制，子进程无需重复加载资源，快速孵化。

    理解 ART 虚拟机的启动流程，不仅能帮助开发者深入掌握 Android 系统的底层运行机制，更能为应用性能优化、崩溃排查、底层开发提供核心理论支撑。
