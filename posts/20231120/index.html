    搭建linux2.6.26内核编译调试环境

    概述

    关于linux内核的基本经典书籍都是基于linux 2.4或者2.6版本，为了更好的阅读我们需要搭建一个老版本的linux内核调试环境。 本文以linux 2.6.26版本的内核为例，介绍如何使用docker搭建一个适用于mac/linux的linux内核32位i386编译调试环境。 其他版本的内核都可以用类似方式搭建，关键是要选择一个合适的基础镜像，例如linux 2.6.26版本用ubuntu14.04编译更为合适。

    运行ubuntu 14.04

    docker的基本使用不赘述，首先我们拉取ubuntu 14.04 i386镜像，并创建相应的容器运行起来

    1
2
3

    docker pull i386/ubuntu:14.04
docker run --name ubuntu -itd i386/ubuntu:14.04 /bin/bash
docker exec -it ubuntu /bin/bash

    安装编译调试需要的工具

    1
2
3
4
5

    apt-get install -y build-essential
apt-get install -y libncurses5-dev
apt-get install -y git
apt-get install -y qemu
apt-get install -y gdb

    下载linux2.6.26源代码

    1

    git clone --branch v2.6.26 --depth 1 https://github.com/torvalds/linux /home/linux

    修复源码编译错误

    直接编译linux内核代码会报错，需要做 几处改动 下载上述文件，使用 git apply 打上补丁就行了。

    编译内核

    由于docker容器共享宿主系统的架构，而我们希望编译i386的内核，所以编译的时候需要指定 ARCH=i386。

    1
2
3
4

    make ARCH=i386 defconfig
# 如有需要可自定义配置
make ARCH=i386 meneconfig
make ARCH=i386 V=1

    等待一段时间，编译成功后会看到两个输出文件:

    • vmlinux: 包含所有内核代码的elf文件，可用于调试内核
    • bzImage: 可用于引导的内核二进制文件

    制作根文件系统

    其实这样就可以开启调试linux内核了，只是内核运行起来后，由于没有设置根文件系统，也没有指定init程序，内核跑起来后无法创建init进程。 有很多工具可以用来制作根文件系统，例如busybox, buildroot等，当前我们采用最简单的一种方式，自己创建一个可执行的init文件，然后使用cpio打包一个最小的initrd。

    先创建一个简单的init可执行文件，源代码

    1
2
3
4
5
6
7
8
9

    #include <stdio.h>
#include <unistd.h>

int main() {
  while (1) {
    printf("hello, I'm init\n");
    sleep(1);
  }
}

    编译成init可执行文件

    1

    gcc -static -o init init.c

    创建系统启动必要的设备文件，使用cpio打包成initrd压缩文件

    1
2
3
4
5
6
7
8

    mkdir rootfs
cp init rootfs
cd rootfs
mkdir dev
mknod dev/console c 5 1
mknod dev/ram b 1 0

find . | cpio -o -H newc | gzip > ../myinitrd.img

    使用qemu运行内核

    1

    qemu-system-i386 -kernel /home/linux/arch/x86/boot/bzImage -initrd /home/initrd/myinitrd.img -append "root=/dev/ram init=/init console=ttyS0" -nographic

    可以看到内核运行起来后，会执行init程序，每1s会输出一句hello
